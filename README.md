# RBX-ST Telegram Bot v2

Новая версия бота с упором на интерфейс и inline-навигацию:
- приветствие через GIF/animation
- пользовательское соглашение с кнопкой принятия
- вся навигация через inline-кнопки и callback
- единое редактируемое меню вместо мусора в чате
- профиль, баланс, наличие, история
- покупка через пошаговый flow
- базовая админка
- настройка стартовой GIF прямо из админки
- поддержка тестовой привязки `TEST_SITE_USER_ID`

## Что уже готово
- красивый каркас Telegram UX
- локальное хранение принятия соглашения и GIF-настройки бота
- интеграционный слой под сайт
- админ-панель, которая **готова синхронизироваться с сайтом**, если backend отдаст нужные admin endpoints

## Что важно
Этот архив честно сделан как **рабочий v2-каркас**, но часть админ-функций зависит от backend сайта.

У тебя уже видно реальные публичные маршруты сайта:
- `/api/auth/me`
- `/balance`
- `/stock`
- `/tx`
- `shop_config`

Поэтому в `.env.example` они уже выставлены по умолчанию.

## Что потребуется на стороне сайта для полной админки
Чтобы из бота менять цены, остатки и баланс пользователей, backend сайта должен дать такие ручки:
- `GET /api/admin/shop/config`
- `POST /api/admin/shop/config`
- `GET /api/admin/stock`
- `POST /api/admin/stock`
- `GET /api/admin/orders`
- `GET /api/admin/users/find?q=...`
- `POST /api/admin/users/balance`

Если названия другие — просто поменяй переменные `ENDPOINT_*`.

## Запуск на Railway
Start Command:
```bash
python -m app.main
```

## Переменные
Минимум:
```env
BOT_TOKEN=...
API_BASE_URL=https://rbx-store-production-b8d7.up.railway.app
API_SECRET=...
TEST_SITE_USER_ID=1
```
